## How each page is structured

Every file contains, in this order:

1. **Page goal** — what this page must accomplish
2. **Target audience** — who it's written for
3. **SEO strategy** — keyword targeting + intent
4. **URL slug** — final URL path
5. **SEO title** — `<title>` tag
6. **Meta description** — `<meta name="description">`
7. **Full page content** — Hero → Symptoms → Causes → Treatment → Benefits → Eligibility → Procedure → Recovery → FAQs → CTAs
8. **FAQ section** — 10–20 patient questions for featured snippets + FAQ schema
9. **Internal linking strategy** — outbound + inbound link plan
10. **Suggested schema** — JSON-LD blocks (MedicalWebPage, FAQPage, MedicalProcedure, LocalBusiness, Physician)
11. **CTA suggestions** — sticky bar, WhatsApp, form, etc.
12. **Suggested supporting blog topics** — additional cluster ideas to commission

Always position outcomes as pain reduction, improved mobility, slowed degeneration, and delayed surgery where medically appropriate.
